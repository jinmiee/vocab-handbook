# 개발 단어장 (Vocab Handbook for Mobile)

이 프로젝트는 사용자가 로컬에 준비한 엑셀 단어장 파일을 기반으로 빌드 시점에 정적 데이터를 생성하여, 아이폰을 비롯한 모바일 기기에서 앱처럼 쾌적하게 학습할 수 있도록 제작된 **모바일 퍼스트 반응형 PWA(Progressive Web App) 단어장**입니다.

---

## 📱 주요 특징 및 기술 스택

- **프레임워크**: Next.js 14+ (App Router), React, TypeScript
- **스타일링**: Tailwind CSS (모바일 레이아웃 최적화, 넉넉한 글자 크기/여백, 3D 플립 카드 효과)
- **로컬 저장소**: 브라우저 `localStorage`를 활용하여 학습 상태(외운 단어, 헷갈려요/몰라요 대상, 최근 본 단어, 북마크, 데일리 퀴즈 성적 이력)를 관리 (Hydration 에러 차단 설계)
- **Spaced Repetition 복습 시스템**: `외웠어요 / 헷갈려요 / 몰라요` 선택과 퀴즈 정오답에 따라 단어별 다음 복습일, 연속 정답, 오답 횟수를 자동 계산합니다.
- **오늘의 복습 / 신규 단어 감지**: 홈 화면에서 오늘 다시 볼 단어 10개를 바로 시작할 수 있고, `vocab.json` 업데이트 후 새로 추가된 단어만 `NEW`로 표시합니다.
- **학습 기록 관리**: 통계 화면 하단에서 북마크, 오답노트, 외운 단어, 복습 스케줄, 퀴즈 이력 등 `localStorage` 기반 학습 데이터를 JSON으로 내보내고 다시 불러올 수 있습니다.
- **면접 대비 학습**: 단어 상세 화면에서 관련 단어와 예상 면접 질문을 함께 보여주고, 퀴즈는 4지선다 외에도 O/X, 빈칸, 비교형 문제를 제공합니다.
- **PWA (오프라인 학습 지원)**:
  - `public/sw.js`를 이용한 Stale-While-Revalidate 캐싱 전략. 오프라인 상태에서도 이미 방문했던 단어장을 끊김 없이 탐색할 수 있습니다.
  - 모바일 홈 화면 추가(Add to Home Screen) 지원 및 iOS Safari safe-area-inset-bottom(아이폰 홈 바 가려짐 방지) 및 viewport-fit 최적화 완료.

## ✨ 주요 기능 소개

### 1. 홈 대시보드

- 전체 단어 수, 외운 단어 수, 복습 대상 수, 학습 진행률을 한 화면에서 확인할 수 있습니다.
- 오늘 다시 봐야 할 복습 단어를 우선순위대로 보여주고, 바로 복습을 시작할 수 있습니다.
- 단어장 데이터가 업데이트되면 새로 추가된 단어를 `NEW`로 구분해 확인할 수 있습니다.

### 2. 단어장 목록

- 키워드와 한줄설명을 기준으로 단어를 검색할 수 있습니다.
- 시트명 기반 대분류와 학습 상태를 조합해 단어를 필터링할 수 있습니다.
- 검색 결과에서는 검색어가 하이라이트되어 원하는 단어를 빠르게 찾을 수 있습니다.
- 단어 상세 화면에서 뒤로 가기를 눌러도 이전에 보던 목록 위치가 유지되도록 구성되어 있습니다.

### 3. 단어 상세 학습

- 키워드, 한줄설명, 상세설명, 세부분류를 모바일 화면에 맞춰 읽기 좋게 제공합니다.
- `외웠어요 / 헷갈려요 / 몰라요` 상태를 선택하면 학습 기록과 다음 복습일이 자동으로 갱신됩니다.
- 같은 분류나 비슷한 키워드를 가진 관련 단어를 함께 보여줘 연결 학습을 할 수 있습니다.
- 단어별 예상 면접 질문과 핵심 키워드를 확인하고, 면접 연습 화면으로 바로 이동할 수 있습니다.

### 4. 플래시카드 암기

- 단어 앞면과 설명 뒷면을 3D 플립 카드 방식으로 확인할 수 있습니다.
- 카드 학습 후 `알아요 / 헷갈려요 / 몰라요`를 선택하면 복습 스케줄과 오답노트가 함께 갱신됩니다.

### 5. 퀴즈 학습

- 객관식 퀴즈로 단어 이해도를 점검할 수 있습니다.
- 문제 유형은 단어→설명, 설명→단어, 빈칸, O/X, 비교형 문제를 지원합니다.
- 정답과 오답을 즉시 색상으로 피드백하고, 틀린 단어는 오답노트에 자동 반영됩니다.

### 6. 면접 질문 연습

- 기존 단어 데이터의 키워드와 상세설명을 활용해 주관식 면접 질문을 자동 생성합니다.
- 카테고리별, 복습 대상별, 북마크 단어별로 면접 질문을 골라 연습할 수 있습니다.
- 사용자가 직접 적은 답변에서 핵심 키워드 포함 여부를 체크리스트로 보여줍니다. (자동 채점이 아니라 참고용 체크입니다.)
- 모범 답변과 꼬리 질문을 확인한 뒤 `잘 답변함 / 조금 부족함 / 다시 봐야 함`으로 직접 자기 평가를 남길 수 있습니다.
- 자기 평가는 기존 학습 상태와 복습 주기 계산에 연결됩니다.

### 7. 오답노트

- 헷갈리거나 틀린 단어만 따로 모아 다시 볼 수 있습니다.
- 오답 단어만 대상으로 퀴즈를 풀 수 있고, 정답을 맞힌 단어는 오답노트에서 자동으로 정리됩니다.

### 8. 학습 통계와 기록 관리

- 전체 진행률, 분류별 학습률, 오늘 학습량, 최근 7일 학습량, 연속 학습일을 확인할 수 있습니다.
- 오답률이 높은 단어와 취약 분류를 확인해 약한 부분을 중심으로 복습할 수 있습니다.
- 통계 화면 하단에서 학습 기록을 JSON 파일로 내보내거나 다시 불러올 수 있습니다.

### 9. 엑셀 기반 단어장 관리

- `단어장.xlsx`의 여러 시트를 대분류로 읽고, `세부분류 / 키워드 / 한줄설명 / 상세설명` 컬럼을 앱 데이터로 변환합니다.
- `npm run parse-excel:merge`로 기존 단어장은 유지하면서 새 단어만 추가하거나 기존 단어 설명을 업데이트할 수 있습니다.
- 변환 후 터미널과 README 업데이트 기록에서 추가/수정/변경 없음/최종 전체 단어 수를 확인할 수 있습니다.

## ⚡ 실행 및 빌드 방법

### 1. 개발 서버 구동
로컬에서 앱을 검토하고 수정하려면 아래 명령어를 입력합니다.
```bash
npm run dev
```
로컬 호스트 주소(기본: `http://localhost:3000`)로 모바일 크기 브라우저를 띄워 테스트할 수 있습니다.

### 2. 정적 배포 빌드
모든 페이지를 순수 정적 HTML/CSS/JS 파일로 변환하여 `out/` 폴더에 출력합니다. GitHub Pages 또는 Netlify, Vercel 정적 호스팅에 즉시 배포할 수 있는 상태가 됩니다.
```bash
npm run build
```

## 📊 엑셀 데이터 갱신 방법

단어 내용이 바뀌었거나 엑셀 시트에 단어를 추가한 경우 아래 절차대로 진행하세요.

> 원본 엑셀 파일(`*.xlsx`, `*.xls`)은 개인 학습 데이터이므로 `.gitignore`에 의해 Git에 포함되지 않습니다. 이 저장소를 내려받은 사용자는 아래 형식에 맞춰 직접 `단어장.xlsx` 파일을 준비해야 합니다.

1. **엑셀 파일 준비**: 프로젝트 루트 디렉토리에 `단어장.xlsx` 파일을 생성하거나 복사해 넣습니다.
   - 파일명은 기본값 기준 `단어장.xlsx`입니다.
   - 시트명은 대분류로 사용됩니다. 예: `공통`, `1.SW공학`, `2.FE`, `3.BE`, `4.CA`, `5.OS`, `6.NW`, `7.DB`, `8.보안`
   - 각 시트의 첫 번째 행은 아래 컬럼명을 사용합니다.

   | 세부분류 | 키워드 | 한줄설명 | 상세설명 | 리뷰 | 출처 |
   | --- | --- | --- | --- | --- | --- |
   | AI·머신러닝 | Agent | LLM이 목표를 달성하기 위해 도구를 선택하고 여러 단계를 수행하는 AI 실행 단위 | Agent의 구성 요소, 사용 예시, 주의점 등을 여러 줄로 작성 | 선택 입력 | 선택 입력 |

   필수 컬럼은 `세부분류`, `키워드`, `한줄설명`, `상세설명`입니다. `리뷰`, `출처`는 엑셀 관리용으로 둘 수 있지만 현재 앱 데이터에는 직접 표시하지 않습니다.
2. **데이터 변환 스크립트 실행**:

   대부분의 경우에는 아래 명령어를 사용하면 됩니다.

   ```bash
   npm run parse-excel:merge
   ```

   이 명령어는 저장소에 들어있는 기본 단어장은 그대로 두고, 내가 만든 `단어장.xlsx`의 단어를 추가합니다. 이미 같은 단어가 있으면 새 내용으로 업데이트합니다.

   예를 들어 기본 단어장이 756개이고, 내 엑셀에 새 단어 20개를 넣었다면 결과는 776개가 됩니다. 내 엑셀에 이미 있던 단어의 설명을 고쳐 넣었다면, 그 단어는 중복으로 추가되지 않고 설명만 갱신됩니다.

   기본 단어장을 버리고 내 엑셀 내용만으로 완전히 새로 만들고 싶을 때만 아래 명령어를 사용하세요.

   ```bash
   npm run parse-excel
   ```

   두 명령어의 차이는 아래와 같습니다.

   | 상황 | 사용할 명령어 | 결과 |
   | --- | --- | --- |
   | 기본 단어장에 내 단어를 추가하고 싶음 | `npm run parse-excel:merge` | 기존 단어 유지 + 내 단어 추가 |
   | 기본 단어장 없이 내 엑셀만 쓰고 싶음 | `npm run parse-excel` | 기존 단어 삭제 후 내 엑셀 기준으로 교체 |

   다른 사람이 이 저장소를 clone해서 자기 단어장을 추가하더라도, 그 변경은 그 사람의 로컬 파일에만 생깁니다. 저장소에 push할 권한이 있는 협업자가 아니라면 원본 저장소에는 직접 반영되지 않습니다. Pull Request를 보내더라도 저장소 소유자가 병합하지 않으면 원본 저장소의 기본 단어장 데이터는 바뀌지 않습니다.

   변환이 끝나면 터미널에 `existing vocab.json`, `parsed from Excel`, `added`, `updated`, `unchanged`, `output total`이 표시됩니다. 이 숫자로 현재 데이터가 몇 개였고, 엑셀에서 몇 개를 읽었고, 몇 개가 새로 추가되거나 수정됐는지 확인할 수 있습니다. 같은 내용은 README의 **단어장 업데이트 기록** 표에도 자동으로 남습니다.
3. **단어 ID 생성 원리**:
   단어의 ID는 단순히 행 번호(index)가 아니라, `시트명(대분류) + 단어(키워드)`의 조합을 MD5로 해싱한 문자열로 구성됩니다. 따라서 엑셀의 데이터 순서가 바뀌거나 행이 추가되더라도, **기존 단어의 ID가 깨지지 않아 사용자의 북마크 및 오답노트 기록이 안전하게 보존**됩니다.

## 📱 iPhone Safari 홈 화면에 추가하여 앱처럼 쓰는 방법

1. 아이폰의 **Safari** 브라우저를 열고 서비스가 배포된 주소에 접속합니다.
2. 하단 도구바 중앙의 **공유 버튼**(위로 향한 화살표 모양 아이콘)을 누릅니다.
3. 공유 옵션 목록에서 스크롤을 내려 **'홈 화면에 추가'**를 선택합니다.
4. 이름을 설정한 뒤 우측 상단의 **'추가'** 버튼을 누릅니다.
5. 홈 화면에 생성된 단어장 앱 아이콘을 누르면, 브라우저 상단 주소창이 사라지고 독립된 풀스크린 네이티브 앱처럼 동작합니다.

- [ ] **학습 상태**: 단어 목록에서 임의의 단어를 누른 뒤 상세 페이지에서 `외웠어요 / 헷갈려요 / 몰라요`를 탭하고, 홈 화면의 진행률 게이지와 통계 탭에서 수치가 실시간 동기화되는지 확인.
- [ ] **복습 스케줄**: 상세 페이지 또는 플래시카드에서 `외웠어요 / 헷갈려요 / 몰라요`를 선택했을 때 다음 복습일과 오늘의 복습 단어 목록이 갱신되는지 확인.
- [ ] **학습 기록 관리**: 통계 화면 하단에서 기록 내보내기 후 기록 초기화, 기록 불러오기를 실행했을 때 북마크/오답/복습 스케줄이 복원되는지 확인.
- [ ] **신규 단어 감지**: 단어장 데이터가 추가된 뒤 기존 학습 상태는 유지되고 목록 화면에서 신규 단어에 `NEW` 배지가 보이는지 확인.
- [ ] **검색 및 필터**: 목록 화면에서 검색어를 입력했을 때 키워드와 한줄설명 영역에 노란색 형광펜 하이라이팅이 잘 입혀지는지, 카테고리 필터와 즐겨찾기 필터가 올바르게 차단/작동하는지 확인.
- [ ] **면접 질문 연습**: 퀴즈 화면의 `면접 질문` 탭에서 카테고리/연습 대상을 고른 뒤 주관식 답변을 입력하면 핵심 키워드 체크, 모범 답변, 꼬리 질문, 자기 평가가 동작하는지 확인.
- [ ] **상세 학습 확장**: 상세 페이지에서 관련 단어, 면접 질문, 다음 복습 추천 정보가 표시되는지 확인.
- [ ] **암기 플래시카드**: 카드를 터치했을 때 3D 회전 애니메이션이 부드럽게 돌아가며 정답(뒷면)이 나타나는지, '알아요/헷갈려요/몰라요' 버튼을 누르면 다음 카드로 자연스럽게 넘어가고 복습노트에 저장되는지 확인.
- [ ] **퀴즈**: 4지선다, 설명→단어, 빈칸, O/X, 비교형 문제에서 초록색(정답)/빨간색(오답) 즉시 피드백이 노출되는지, 퀴즈 종료 시 리포트가 정상 생성되고 틀린 단어들만 오답노트에 반영되는지 확인.
- [ ] **오답노트 퀴즈**: 오답노트에서 '오답 단어로 퀴즈 풀기'를 실행했을 때, 퀴즈에서 정답을 맞힌 단어는 오답노트 목록에서 자동으로 소거되는지 확인.
- [ ] **학습 리포트**: 통계 화면에서 오늘 학습량, 최근 7일 학습량, 연속 학습일, 취약 분류, 오답 TOP 단어가 학습 기록 기준으로 표시되는지 확인.
- [ ] **PWA 동작**: 오프라인 모드(크롬 개발자 도구의 Network -> Offline) 상태로 설정하고 앱 새로고침 시에도 기본 화면 및 단어 데이터 조회가 가능한지 확인.
